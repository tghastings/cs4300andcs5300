\section{Detailed Requirements}

\subsection{Individually - Delivered Peer Feedback (12 pts)}
Each team member must deliver constructive peer feedback to teammates.

\subsection{Project in GitHub (0 pts)}
Your project must continue to be maintained in GitHub (no points assigned this sprint as this is now expected).

\subsection{Features with Tests (30 pts)}
All features developed during this sprint must have comprehensive tests. This requirement carries increased weight to emphasize the importance of well-tested features.

\subsection{80\% Test Coverage Metrics (13 pts)}
Your test suite must maintain at least 80\% code coverage.

\subsection{CI Pipeline (37.5 pts total)}
Your continuous integration pipeline must include:
\begin{itemize}
    \item \textbf{AI Code Review (2.5 pts):} Using OpenAI Platform for each Pull Request
    \item \textbf{Automated Tests (2.5 pts):} Running automated tests for each commit
    \item \textbf{Coverage Reports (2.5 pts):} Reports test coverage metrics in console for each commit
    \item \textbf{\textcolor{newitem}{NEW:} Linting (15 pts):} PyLint and/or Flake8 linting scans in CI pipeline with reports
    \item \textbf{\textcolor{newitem}{NEW:} Dependency Scans (15 pts):} Dependabot dependency vulnerability scans in CI pipeline with reports
\end{itemize}

\subsection{\textcolor{newitem}{NEW:} Custom Domain (5 pts)}
Your application must be deployed to a production environment with a custom domain name. No subdomains are allowed unless you own the top-level domain.

\subsection{CD Pipeline (2.5 pts)}
Code must continually be deployed to production environment using GitHub Actions when merged into main.
